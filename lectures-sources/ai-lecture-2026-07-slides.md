## Що таке комп'ютерний зір?

**Мета:** навчити комп'ютер «розуміти» зображення — що зображено, де знаходиться, що відбувається

### Застосування

| Галузь | Приклади |
|---|---|
| Робототехніка | Навігація, маніпуляція об'єктами |
| Медицина | Діагностика раку, аналіз рентгенів |
| Автономне водіння | Розпізнавання пішоходів, знаків, смуг |
| Доступність | Навігаційні додатки для незрячих |
| Ігри | Класифікація спрайтів, аналіз сцен, QA-тестування |

> **Ключова ідея:** комп'ютер «бачить» лише матрицю чисел — завдання CV перетворити ці числа на розуміння

---

## Зображення — це числа

Для комп'ютера зображення — це **тривимірна матриця** чисел від 0 до 255:

- **Чорно-біле:** матриця $H \times W$ (один канал)
- **Кольорове (RGB):** три матриці $H \times W \times 3$ — по одному каналу на червоний, зелений, синій

**Приклад:** фото 1080×1080 RGB = $1080 \times 1080 \times 3 = 3\,499\,200$ чисел

### Задачі комп'ютерного зору

- **Класифікація:** визначити, що на зображенні (кіт, авто, гравець)
- **Детекція об'єктів:** знайти де саме (рамка навколо об'єкта)
- **Сегментація:** визначити клас для кожного пікселя

---

## Чому не звичайна нейромережа?

### Проблема повнозв'язних мереж (fully connected)

Якщо подати зображення як вектор пікселів у звичайну нейромережу:

1. **Втрачається просторова структура** — мережа не знає, що сусідні пікселі пов'язані
2. **Занадто багато параметрів** — для зображення 224×224×3 перший шар матиме мільйони ваг

### Проблема ручного підходу

Визначати ознаки вручну (наприклад, "знайди кола для розпізнавання колес") не працює через:
- зміну ракурсу, масштабу, освітлення
- деформації та перекриття об'єктів
- варіації всередині одного класу

> **Рішення:** навчити мережу самостійно знаходити **ієрархію ознак** — від простих (краї, плями) до складних (очі, обличчя)

---

## Операція згортки (convolution)

**Згортка** — це "розумне ковзне вікно", яке шукає локальні патерни:

### Як працює:
1. Беремо маленьку матрицю ваг — **фільтр** (наприклад, 3×3)
2. **Прикладаємо** до кожної ділянки зображення
3. **Поелементно множимо** значення фільтра на пікселі та **сумуємо** — отримуємо одне число
4. **Зсуваємо** фільтр і повторюємо — отримуємо **карту ознак** (feature map)

### Три принципи згортки:
- **Локальні зв'язки:** кожен нейрон "бачить" лише маленький патч
- **Спільні ваги:** один фільтр застосовується до всього зображення
- **Множинні фільтри:** кожен фільтр шукає свою ознаку (краї, текстури, кути)

> Різні фільтри виявляють різне: загострення, виявлення країв, розмиття

---

## Архітектура CNN

**CNN (Convolutional Neural Network)** — мережа, що чергує три типи операцій:

### 1. Згортка (Convolution)
Застосовує фільтри → генерує карти ознак

### 2. Нелінійність (ReLU)
$g(z) = \max(0, z)$ — обнулює від'ємні значення, дозволяє моделювати складні залежності

### 3. Пулінг (Pooling)
Зменшує розмір карти ознак (наприклад, MaxPool 2×2: з кожного блоку 2×2 залишає максимум)
- Зменшує кількість обчислень
- Надає **інваріантність до зсувів** — об'єкт розпізнається незалежно від позиції

### Повна архітектура для класифікації:
`Вхід → [Згортка → ReLU → Пулінг] × N → Розгортання → Повнозв'язний шар → Softmax`

---

## CNN у коді (PyTorch)

```python
import torch
import torch.nn as nn

model = nn.Sequential(
    # Перший згортковий блок
    nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2, stride=2),

    # Другий згортковий блок
    nn.Conv2d(in_channels=32, out_channels=64, kernel_size=3),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2, stride=2),

    # Класифікатор
    nn.Flatten(),
    nn.Linear(64 * 6 * 6, 1024),
    nn.ReLU(),
    nn.Linear(1024, 10),   # 10 класів
)
```

### Ключові параметри `Conv2d`:
- `in_channels` — кількість вхідних каналів (3 для RGB)
- `out_channels` — кількість фільтрів (= кількість карт ознак)
- `kernel_size` — розмір фільтра (3 = матриця 3×3)
- `stride` — крок зсуву фільтра

---

## Ієрархія ознак у глибоких CNN

Кожен наступний згортковий шар виявляє **дедалі складніші патерни:**

| Шар | Що виявляє | Приклади |
|---|---|---|
| Conv 1 | Прості ознаки | Краї, лінії, плями |
| Conv 2 | Середні ознаки | Текстури, кути, прості форми |
| Conv 3+ | Складні ознаки | Очі, вуха, колеса, вікна |
| Глибші шари | Об'єкти | Обличчя, автомобілі, будинки |

### Навчання
- Ваги фільтрів **не програмуються вручну** — мережа вивчає їх під час тренування
- Використовується звичний **градієнтний спуск** з backpropagation
- Функція втрат — **cross-entropy** для класифікації

> Це і є суть глибокого навчання: автоматичне вивчення корисних представлень даних

---

## Застосування CNN

### Класифікація
Визначення класу всього зображення (кіт/собака, тип текстури, жанр скріншота)

### Детекція об'єктів
- **R-CNN**: знаходить ~2000 "кандидатів", класифікує кожного — повільно
- **Faster R-CNN**: вчить пропонувати регіони нейромережею — швидко

### Семантична сегментація
Кожному пікселю присвоюється клас (небо, дорога, автомобіль, пішохід)
- **FCN (Fully Convolutional Network):** downsampling + upsampling, повністю зі згорток

### Автономна навігація
CNN приймає зображення з камери → видає команди керування (кут повороту, швидкість)

### В іграх
- Автоматична класифікація ігрових ассетів та текстур
- QA-тестування: пошук візуальних багів на скріншотах
- Аналіз сцен для процедурної генерації
