Детекція об'єктів: формулювання

Для кожного об'єкта модель передбачає:


Координати рамки

(x, y, w, h)

Клас об'єкта

c ∈ {1, ..., C}

Впевненість

p(c) ∈ [0, 1]


Складність: десятки/сотні об'єктів різного розміру, з перекриттями, на складному фоні — за один прохід мережі

Еволюція підходів до детекції

Двоетапні (Two-stage)

  • R-CNN (2014): Selective Search + CNN
  • Fast R-CNN: CNN один раз
  • Faster R-CNN: Region Proposal Network

~5 FPS — точно, але повільно

Одноетапні (One-stage)

  • YOLO: сітка S×S, один прохід
  • SSD: multi-scale feature maps
  • RetinaNet: Focal Loss

400+ FPS — швидко і точно

YOLO: від v1 до v11

mAP@50:95 на COCO: v1 (63%) → v8 (53.9% mAP@50:95) при 400+ FPS

Метрики: IoU та mAP

IoU (Intersection over Union)

IoU = |Bpred ∩ Bgt| / |Bpred ∪ Bgt|

IoU ≥ 0.5 → об'єкт «знайдено»

mAP (mean Average Precision)

  • Для кожного класу — крива precision-recall
  • AP = площа під кривою
  • mAP = середнє по класах
  • mAP@50:95 — основний бенчмарк

Три рівні сегментації

Семантична — класи; Інстансна — окремі об'єкти; Паноптична — все разом

U-Net: encoder-decoder з skip connections

Encoder зменшує → витягує ознаки. Decoder відновлює → локалізує межі. Skip connections зберігають деталі.

Vision Transformer (ViT)

Зображення → патчі 16×16 → embedding + позиції → Transformer encoder → класифікація/детекція/сегментація

Swin Transformer та DETR

Swin Transformer (2021)

  • Локальні вікна → лінійна складність
  • Shifted windows між шарами
  • Ієрархічне представлення
  • Новий SOTA backbone для CV

DETR (2020)

  • End-to-end детекція
  • Без anchor boxes, без NMS
  • Object queries → predictions
  • Hungarian matching

Для ігор: глобальний контекст, гнучкість роздільності, природне поєднання з мовними моделями

CLIP: зір + мова

400M пар «зображення + текст» → спільний embedding-простір → zero-shot класифікація

CLIP в ігровій розробці

Семантичний пошук ассетів: художник пише «mossy stone wall in forest» → знаходить релевантні текстури з бібліотеки на 50K ассетів
Автокласифікація: жанр скріншоту, настрій сцени, наявність елементів — без спеціального тренування
Генерація з тексту: CLIP + Stable Diffusion = концепт-арт або текстура за текстовим описом
Стилістична консистентність: перевірка, чи новий ассет відповідає арт-стилю гри через embedding-відстань

Швидкість: що працює в реальному часі?

Легкі моделі (YOLO-nano, Depth Anything-S) → реалтайм. Важкі (SAM, Swin-L) → інструменти розробки.

Оптимізація для ігрових рушіїв

Формати

  • ONNX — універсальний обмін
  • TensorRT — NVIDIA, FP16/INT8
  • Core ML — Apple Neural Engine

Інтеграція

  • Barracuda — ONNX в Unity
  • NNE — ONNX в Unreal Engine 5
  • PyTorch → torch.export → ONNX

PyTorch → torch.export → ONNX → TensorRT → Ігровий рушій (C++) → GPU інференс (< 5 мс)

Повний пайплайн розуміння сцени

YOLO + DeepLab + Depth Anything → fusion → 3D-позиції, прохідні зони, контекст → ігрова логіка

Ключові висновки

  1. YOLO — реалтайм детекція для NPC-зору, QA, античіту
  2. Сегментація (U-Net → SAM) — попіксельне розуміння для процедурної генерації та AR
  3. Vision Transformers — глобальний контекст замість локальних згорток
  4. CLIP — зв'язок зору та мови для zero-shot і генерації
  5. Depth Anything — 3D з одного фото для AR, нормалей, пост-обробки
  6. CV в іграх = всередині + у розробці + навколо гри
